# 懸念点と対策（Issue #68ベース）

## 1. 初期の過疎問題
- リスク: ユーザーが少ないと価値が立ち上がらない
- 対策: エリア限定ローンチ + 初期コミュニティ施策

## 2. モデレーション負荷
- リスク: 詐欺・差別・荒らし対応が運用負担になる
- 対策: 通報/凍結/本人確認フローを先に設計

## 3. 信頼設計の難しさ
- リスク: 匿名性が高いと取引事故が増える
- 対策: バッジ、評価履歴、取引証跡の導入

## 4. 多言語品質リスク
- リスク: 翻訳誤差でトラブルが発生
- 対策: 定型文 + 重要項目の再確認UI

## 5. 法務・規約リスク
- リスク: 売買/求人/住居カテゴリで規制が絡む
- 対策: カテゴリを段階導入、法務レビューを先行

## 6. マネタイズの遅さ
- リスク: コミュニティ先行で収益化が遅れる
- 対策: 協賛・プレミアムの早期仮説検証
